Prefix Handle Example

The figure below shows the record of the prefix Handle 0.NA/10.1009.

The table below describes the elements of the 0.NA/10.1009 record.

 

 

Type

Index

Description

1

HS_ADMIN

100

RA or registrant's prefix administrator: can create Handles (DOI names) under prefix 10.1009.
This administrator is identified1) by 200:0.NA/10.1009 which points to the index 200 in this identifier (HS_VLIST).

2

HS_ADMIN

101

Prefix administrator of the DOI technical support service provider: can create and delete Handles (DOI names) under prefix 10.1009; can create and delete prefixes deriving from 10.1009.

3

HS_SERV

1

Points to the 10.SERV/CROSSREF Handle which locates the Local Handle Service (LHS) sites managing the DOI names under prefix 10.1009.

4

HS_VLIST

200

List of administrators (list of public keys).

5

EMAIL

3

Email address.

6

HS_SIGNATURE

400

Signature over the prefix Handle.

1) An administrator is identified in the Handle System by a public key stored in a Handle Record (in this example, the public key is at index 200 of 0.NA/10.1009 record). User authentication is performed by a Handle System service through a PKI challenge-response.
